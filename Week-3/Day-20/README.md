## Problem Statement

Build an ESG (Environmental, Social, and Governance) platform that goes beyond compliance tracking.

**Challenge:** Most ESG platforms are just reporting tools. They don't engage employees.

**Our Solution:** Transform ESG tracking into an employee engagement platform using gamification.

---

## My Contribution: Social & Gamification Module

### Core Concept

Instead of treating ESG as compliance, reward employees for actively participating in sustainability initiatives.

**Inspiration:** Microsoft Learn Student Ambassador Program progression system.

**Goal:** Create continuous motivation through achievements, progression, and healthy competition.

---

### Features I Designed & Built

#### 1. CSR Activities
- Admins create volunteering and sustainability events
- Employees browse, register, upload proof of participation
- Admin approval system
- Automatic XP rewards upon approval

#### 2. Sustainability Challenges
- Challenge creation system with difficulty levels
- Progress tracking
- Evidence upload and verification
- XP rewards for completion

#### 3. Experience Points (XP) System
- Tracks employee participation across all activities
- Earned through approved CSR activities and completed challenges
- Foundation for progression system

#### 4. Badge System
- Automatic badge unlocking based on achievement rules
- Milestone-based rewards (XP thresholds, challenge completion)
- Visual recognition of accomplishments

#### 5. Rewards Store
- Point redemption system
- Stock validation before redemption
- Point deduction upon successful redemption

#### 6. Leaderboard
- Dynamic ranking based on XP
- Department-level competition
- Encourages healthy competition between teams

---

### Employee Journey I Designed

Login → Personal Dashboard → Browse CSR Activities → Join Activity
→ Upload Proof → Admin Verification → Earn XP
→ Participate in Challenges → Track Progress → Unlock Badges
→ Climb Leaderboard → Redeem Rewards


---

### Technical Architecture

**Modular design to avoid merge conflicts during hackathon:**
- Separate frontend pages
- Independent backend controllers
- Dedicated routes and services
- Isolated models and database tables
- Easy integration with main application

**Why this mattered:**
- Team could work in parallel
- No merge conflicts during 8-hour sprint
- Scalable module design
- Clean integration
