## Key Learning: Don't Over-Optimize

**The realization:**
I was stuck trying to find O(n) solution when O(n²) was optimal for the problem constraints.

**What I learned:**
- Read constraints carefully
- Sometimes O(n²) IS the right answer
- Don't overthink before understanding the problem
- 70% understanding + move forward > 100% stuck

**This changes how I approach problems:** Speed through with good understanding, revisit gaps later (not perfect first time).
