## Key Learning: Iteration vs Recursion

**Same problems, different approach:**

| Problem | Iterative (Previous) | Recursive (Today) |
|---------|---------------------|-------------------|
| Reverse String | 10 mins (Day 9) | 45 mins |
| Reverse Linked List | 20 mins (Day 15) | 1hr 20 mins |

**Why recursion takes longer:**
- Need to trust the recursion (hard mentally)
- Visualizing call stack is confusing
- Base case + recursive case both need to be correct
- Harder to debug when wrong

**But why learn it:**
- Some problems are naturally recursive (trees, backtracking)
- Builds different problem-solving muscle
- Industry interviews often test recursion
